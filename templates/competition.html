MaxCLIP Competition

Welcome to the MaxCLIP competition page! Here you can find all the details about our latest competition, including rules and deadlines.

Rules and Guidelines

Below are the rules and guidelines for participating in the competition:

  • Eligibility: The competition is open to individuals and teams from all backgrounds, including university students, researchers, and industry professionals.
  • Leaderboards:
    • Constrained leaderboard: our main leaderboard consisting of submissions that only make permitted modifications to the codebase (see Permitted Modifications below).
    • Unconstrained leaderboard: a free-for-all leaderboard containing evaluations of any CLIP model trained in any fashion.
    • The results of each submission will be published on a public leaderboard and will be updated weekly.
  • Prohibited Actions (Constrained Submission):
    • Participants may not use any pre-trained models in their submission.
    • Tampering with model architecture or any other fixed code (for a constrained submission).
    • Tampering with or bypassing the competition's fixed evaluation or training procedure is strictly prohibited and will result in an automatic disqualification.
  • Disqualification: Any team found to be in violation of the rules will be disqualified from the competition.

Submission Rules

  • Each competitor is limited to one submission every two weeks.
  • Submissions must include:
    • The modified code in the provided editable folder.
    • A configuration file with hyperparameters.
  • Submissions must be a single ZIP file.
  • (Constrained) submissions may not include any modifications to frozen components (e.g., dataset, model architecture, etc.) unless explicitly permitted.
  • In case of a technical failure (e.g., bug), the competitor will be notified and allowed to submit a fix within 48 hours.

Dataset

We will release small (1M), medium (10M), and large (100M) datasets which are randomly-chosen image/text pairs from DataComp. Since we wish to measure various methods' scaling laws with respect to dataset size, our small dataset is a subset of the medium dataset, which in turn is a subset of the large dataset.

Evaluation

We will use the publicly-available DataComp evaluation procedure to evaluate submissions. As evaluation metrics, we will publish the DataComp Average Score and ImageNet Zero-Shot Accuracy of each submission. Each submission will be evaluated using the same fixed test dataset and hardware environment. We will evaluate submissions as soon as we can and release leaderboard updates on a weekly basis.

Permitted Modifications

In the constrained track, the following modifications to the codebase are permitted:

  • Loss function or loss computation.
  • Parameter update (includes u and Î³ update for SogCLR).
  • Hyperparameters may be tuned by competitors and submitted along with their code in a custom config file.

A folder with code that competitors are allowed to edit will be provided; everything else (except a configuration file containing hyperparameters) is frozen.

Frequently Asked Questions

Here are some common questions and answers about the competition:

What is the competition about?

The competition aims to explore and optimize the performance of CLIP models on varying scales of datasets.

How can I submit my work?

Submissions must be a single ZIP file including the modified code in the provided editable folder and a configuration file with hyperparameters.

What datasets will be used?

We will use small (1M), medium (10M), and large (100M) datasets randomly chosen from DataComp.
